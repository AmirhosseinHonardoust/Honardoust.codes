Coffee Shop Profit Predictor

A SQL + Python retail analytics project for predicting monthly coffee-shop profit using engineered location features, ElasticNet regression, diagnostics, feature importance, and candidate-site scoring.

EXP-006 / question

Technical question

Can SQL feature engineering and interpretable regression support better coffee-shop site selection decisions?

EXP-006 / method

Method and workflow

  1. Load training and candidate location CSV files into SQLite.
  2. Create SQL feature views for demand, competition, affordability, weekend traffic, and promotion effects.
  3. Train an ElasticNet regression model with standardized preprocessing.
  4. Evaluate with R², MAE, actual-vs-predicted plots, residual diagnostics, and feature importance.
  5. Score candidate locations and export prediction artifacts for decision review.

Business output

EXP-006 / limitations

Limitations and honesty check

  • The dataset is simulated, so real deployment would need actual transaction, location, and market data.
  • Linear regularized regression is interpretable but may miss nonlinear effects between foot traffic, rent, and events.
  • Profit prediction should include uncertainty estimates and sensitivity analysis before business use.

EXP-006 / next

Next improvements

  • Add nonlinear models such as Random Forest, Gradient Boosting, or GAMs for comparison.
  • Add confidence intervals or prediction uncertainty for candidate sites.
  • Create a small Streamlit dashboard for site ranking and what-if analysis.
  • Validate the feature assumptions against real retail data.
