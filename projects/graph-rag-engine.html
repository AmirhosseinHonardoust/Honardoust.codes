Graph-RAG Engine

An explainable retrieval-augmented generation system that combines graph intelligence, vector search, citations, a FastAPI backend, a Streamlit interface, FAISS, and an in-memory knowledge graph.

EXP-001 / question

Technical question

Can retrieval paths make RAG answers more explainable, grounded, and easier to inspect?

EXP-001 / method

Method and workflow

  1. Build a document/chunk/concept knowledge graph.
  2. Use FAISS for dense semantic retrieval.
  3. Expand retrieval with graph relationships and PageRank-style weighting.
  4. Return answer text with citations and reasoning paths.
  5. Expose the workflow through FastAPI endpoints and a Streamlit UI.

EXP-001 / limitations

Limitations and honesty check

  • The graph layer is in-memory, so production persistence would need Neo4j or another graph database.
  • The project is positioned as an MVP, so deployment, authentication, monitoring, and feedback-based evaluation would be future work.
  • Answer quality should be evaluated with retrieval metrics, citation accuracy checks, and out-of-domain documents.
